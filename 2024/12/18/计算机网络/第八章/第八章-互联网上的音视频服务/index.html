《计算机网络（谢希仁）》- 第八章 互联网上的音视频服务

8.1 概述

多媒体数据传输的特点：

  • “边传输，边播放”
  • 信息量往往很大
  • 对传输时延和时延抖动有较高要求

多媒体分组传输过程：

  • 发送时是等时的
  • 到达接收端时变成非等时
  • 接收端处理流程：
    1. 等待缓存中分组达到一定数量
    2. 以恒定速率按顺序还原播放
    3. 产生播放时延，但可以消除时延抖动

时延敏感数据的处理原则：

  • 宁可丢失少量分组，也不要接收太晚到达的分组
  • 典型应用：视频会议

互联网音频/视频服务的三种类型：

  1. 流式存储音频/视频（流媒体）：
    • 边下载边播放
  2. 流式实况音频/视频：
    • 边录制边发送
    • 边下载边播放
  3. 交互式音频/视频：
    • 互联网电话
    • 视频会议等

8.2 流式存储音频/视频

8.2.1 具有元文件的万维网服务器

流媒体特点：

  • 可以边下载边播放
  • 不能存储在硬盘上成为用户文件

8.2.2 媒体服务器

基本概念：

  • 又称流式服务器
  • 更好地支持流式音频和视频传送

传输协议选择：

  • TCP：能保证播放质量，但滞后时间长
  • UDP：适用于实时流式音频/视频文件传送

8.2.3 实时流式协议RTSP

RTSP协议特点：

  • 为流式传送过程增加更多功能
  • 本身不传送数据
  • 仅使媒体播放器能够控制多媒体流的传送

8.3 交互式音频/视频

8.3.1 IP电话概述

IP电话的定义：

  • 通过 IP 网络进行的交互式多媒体实时通信
  • 包括视频通话、即时传信IM（如 QQ）等

IP电话通信质量的决定因素：

  1. 通话双方端到端的时延和时延抖动
  2. 话音分组的丢失率

8.3.2 IP电话所需要的几种应用协议

8.3.3 实时运输协议RTP

RTP的特点：

  • 采用无连接的 UDP 协议
  • 为实时应用提供端到端运输
  • 不提供任何服务质量保证
  • 可看作 UDP 之上的运输层子层协议

工作原理：

  • 将多媒体数据块封装成 RTP 分组
  • 装入运输层的 UDP 数据报

8.3.4 实时运输控制协议RTCP

RTCP的特点：

  • 与 RTP 配合使用
  • 采用 UDP 服务
  • 不对音频/视频分组进行封装

主要功能：

  1. 服务质量的监视与反馈
  2. 媒体间的同步
  3. 多播组中成员的标志

8.3.5 H.323

基本概念：

  • 信令是通信双方为建立呼叫连接和控制而传送的专门信息
  • IP 电话有两套信令标准：H.323 和 SIP
  • H.323 是一组协议，功能多且复杂

H.323标准的四个构件：

  1. H.323 终端
  2. 网关
  3. 网闸
  4. 多点控制单元 MCU

8.3.6 会话发起协议SIP

SIP的特点：

  • 比 H.323 简单
  • 只涉及 IP 电话所需的信令和服务质量
  • 使用文本方式的客户服务器协议

SIP系统的两种构件：

  1. 用户代理：
    • 用户代理客户
    • 用户代理服务器
  2. 网络服务器：
    • 代理服务器
    • 重定向服务器

SIP地址特点：

  • 格式灵活
  • 可以是电话号码、电子邮件、IP 地址等

8.4 改进”尽最大努力交付”的服务

8.4.1 使互联网提供服务质量

服务质量（QoS）：

  • 是服务效能的总效果
  • 决定用户对服务的满意程度

服务质量的性能指标：

  • 可用性
  • 差错率
  • 响应时间
  • 吞吐量
  • 分组丢失率

提高服务质量的措施：

  1. 分类：区分服务
  2. 管制
  3. 调制
  4. 呼叫接纳
  5. 加权公平排队

8.4.2 调度和管制机制

8.4.3 综合服务IntServ与资源预留协议RSVP

综合服务 IntServ：

  • 可对单个应用对话提供服务质量保证
  • 定义了两类服务：
    • 有保证的服务
    • 受控负载的服务

IntServ的四个组成部分：

  1. 资源预留协议 RSVP
  2. 接纳控制
  3. 分类器
  4. 调度器

8.4.4 区分服务DiffServ

区分服务 DiffServ（DS）的特点：

  • 在路由器中增加区分服务功能
  • IP 数据报中有区分服务字段
  • 利用 DS 字段提供不同等级服务质量
  • 将复杂性放在 DS 域的边界结点中
  • 使 DS 域内路由器工作尽可能简单
