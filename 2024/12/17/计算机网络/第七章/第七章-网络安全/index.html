7.1 网络安全问题概述

7.1.1 计算机网络面临的安全性威胁

计算机网络上的通信面临的威胁可以分为两大类：

  • 被动攻击：如截获
  • 主动攻击：如中断、篡改、伪造

7.1.2 安全的计算机网络

安全的计算机网络应该达到：

  • 保密性
  • 端点鉴别
  • 信息的完整性
  • 运行的安全性

7.1.3 数据加密模型

密码学的两个组成部分：

  • 密码编码学（设计密码体制）
  • 密码分析学（破解密码）

密码安全性：

  • 无条件安全的密码几乎不存在
  • 大部分密码都是计算上安全的（在一定时间内不可破解）

7.2 两类密码体制

7.2.1 对称密钥密码体制

对称密钥密码体制的特点：

  • 加密密钥与解密密钥相同
  • 保密性仅取决于对密钥的保密
  • 算法是公开的

数据加密标准：

  • DES 是一种对称密钥密码体制
  • 初版的 DES 已不安全
  • 目前采用三重 DES（3DES）
  • 3DES 广泛用于网络、金融、信用卡等系统

7.2.2 公钥密码体制

公钥密码体制的特点：

  • 加密密钥是公开的（公钥）
  • 解密密钥是保密的（私钥）
  • 加密和解密算法是公开的
  • 私钥是用户私有的，对其他人都保密

相对于对称密钥体制的优点：

  1. 不需要考虑密钥分配问题
  2. 提供数字签名功能

重要实现：

  • RSA 体制（RSA 公钥加密算法）是最著名的公钥密码体制
  • 采用了数论中的大数分解问题

注意： 任何加密方法的安全性取决于密钥的长度和攻破密文所需的计算量，而非体制。

7.3 数字签名

数字签名需要提供三点功能：

  1. 报文鉴别：接收者能够核实发送者对报文的签名，从而验证发送者的身份
  2. 报文的完整性：接收者确信收到的数据和发送者发送的数据完全一样
  3. 不可否认：发送者事后不能否认自己对报文的签名

基于公钥密码体制的数字签名实现方法：

  • 发送者使用自己的私钥对报文进行加密
  • 接收者使用该发送者的公钥来解密报文

7.4 鉴别

鉴别是要验证通信对方是自己要通信的对象。

7.4.1 报文鉴别

报文鉴别的基本概念：

  • 用来鉴别报文的完整性
  • 采用了密码散列函数

散列函数的特点：

  1. 输入长度可以很长，输出长度较短且固定（输出称为散列值）
  2. 不同的散列值对应不同的输入，但不同的输入可能得到相同的散列值

密码学中散列函数的重要特点：

  • 找到两个具有相同散列值的不同报文在计算上是不可行的
  • 根据散列值求报文的逆向变换是不可能的

常用的散列算法：

  • 报文摘要 MD5 是一种简单方法
  • 目前广泛使用安全散列算法 SHA-2/3

散列函数的使用方法：

  1. 对报文计算出散列值
  2. 将散列值附加在报文后面
  3. 将附加了散列值的整个报文加密发送
  4. 接收方解密后重新计算报文的散列值
  5. 比较计算得到的散列值与收到的散列值

7.4.2 实体鉴别

报文鉴别与实体鉴别的区别：

  • 报文鉴别：对每个收到的报文都要鉴别发送者
  • 实体鉴别：在整个连接过程中只需鉴别一次

重放攻击：

  • 场景：A 向 B 发送带有身份和口令的加密报文，B 解密后鉴别 A 的身份
  • 问题：C 可能截获 A 的报文并转发给 B，导致 B 误认为 C 是 A

不重数法：

  • 使用不重复的大随机数来解决重放攻击
  • A 向 B 发送不重数（每次会话使用不同的数）
  • C 截获后再发送时，B 能发现重复从而避免被骗

7.5 密钥分配

密钥分配即密钥分发，是密钥管理中最大的问题，密钥必须通过最安全的通路进行分配。

7.5.1 对称密钥的分配

目前常用的密钥分配方式是设立密钥分配中心 KDC。

7.5.2 公钥的分配

公钥分配的安全问题：

  • 公钥不可随意分配
  • 例如：C 截获 A 发给 B 的报文（用 A 的私钥加密并附有 A 的公钥）再转发给 B，B 无法验证公钥归属

解决方案：

  • 将公钥与对应的实体进行绑定
  • 由认证中心 CA 来完成绑定操作
  • 每个实体都有 CA 发来的证书，包含：
    • 公钥信息
    • 拥有者的标识信息
    • CA 的数字签名
  • 任何用户都可从可信渠道获得认证中心 CA 的公钥

7.6 互联网使用的安全协议

网络层、运输层、应用层都有相应的网络安全协议。

7.6.1 网络层安全协议

基本概念：

  • 网络层使用 IPsec 协议族
  • VPN 就是使用了网络层安全协议
  • IPsec 提供了一套加密算法，但不强制使用特定算法

IP安全数据报的两个协议：

  1. 鉴别首部协议（AH）：
    • 提供源点鉴别和数据完整性
    • 不能保密
  2. 封装安全有效载荷协议（ESP）：
    • 提供源点鉴别、数据完整性和保密
    • 比 AH 复杂很多

IPsec数据报特点：

  • 使用 AH 或 ESP 协议的 IP 数据报称为 IPsec 数据报
  • 在 IPv6 中，AH 和 ESP 都是扩展首部的一部分
  • AH 协议包含在 ESP 协议内部

IPsec数据报的工作方式：

  1. 运输方式：

    • 在运输层报文段前后添加控制信息
    • 再加上 IP 首部，构成 IPsec 数据报

  2. 隧道方式（使用最多）：

    • 在原始 IP 数据报前后添加控制信息
    • 再加上新的 IP 首部，构成 IPsec 数据报

注意： 无论哪种方式，IPsec 数据报的首部都不加密（便于路由器识别），只加密数据部分。

7.6.2 运输层安全协议

运输层安全协议概述：

  • 包括安全套接字层 SSL 和运输层安全 TLS
  • SSL 最新版本是 SSL 3.0
  • TLS 是在 SSL 3.0 的基础上设计的

SSL的工作机制：

  • 作用于端系统应用层的 HTTP 和运输层之间
  • 在 TCP 之上建立安全通道
  • 为通过 TCP 传输的应用层提供安全保障

HTTPS协议：

  • 网址中 https 表示使用了 SSL 协议
  • TCP 的 https 端口号是 443
  • http 端口号是 80

SSL提供的三种安全服务：

  1. SSL 服务器鉴别：允许用户证实服务器的身份
  2. SSL 客户鉴别：允许服务器证实客户的身份
  3. 加密的 SSL 会话：
    • 对客户和服务器之间的所有报文加密
    • 检测报文是否被篡改

SSL的工作过程：

  1. 协商加密算法
  2. 服务器鉴别
  3. 会话密钥计算
  4. 安全数据传输

7.6.3 应用层安全协议

PGP的特点：

  • 是完整的电子邮件安全软件包
  • 包括加密、鉴别、电子签名、压缩等技术
  • 综合了现有的加密算法

7.7 系统安全：防火墙与入侵检测

防火墙和入侵检测系统 IDS 构成了系统防御的两层防线。

7.7.1 防火墙

防火墙的基本概念：

  • 是一种特殊编程的路由器
  • 安装在网点和网络其余部分之间
  • 目的是实施访问控制策略
  • 防火墙内的网络称为可信网络

防火墙技术的两类：

  1. 分组过滤路由器：

    • 根据过滤规则对分组执行转发或丢弃
    • 规则基于分组的网络层或运输层首部信息
    • 例如：丢弃所有目的端口号为 23 的进入内部网络的分组

  2. 应用网关：

    • 也称代理服务器
    • 每种网络应用需要一个应用网关
    • 在应用层打开报文检查合法性
    • 例如：万维网缓存是万维网应用的代理服务器

7.7.2 入侵检测系统

入侵检测系统（IDS）的作用：

  • 在入侵开始后及时检测并阻止

IDS的两种类型：

  1. 基于特征的 IDS：

    • 根据已知攻击的标志性特征检测入侵

  2. 基于异常的 IDS：

    • 根据网络流量的统计特性来检测入侵

7.8 一些未来的发展方向

未来发展方向：

  1. 椭圆曲线密码（ECC）与 AES
    • 下一代金融系统使用的加密系统
  2. 移动安全
    • 移动通信所需要的安全机制
  3. 量子密码
